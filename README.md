## 学习笔记：
### 概念：
- **VPN**：一种技术协议，常用于加密数据并保护网络安全。
- **VPS**：Virtual Private Server（虚拟专用服务器），远程的、虚拟化的计算机。
- **机场**：提供节点订阅和管理服务的商业销售形式。
- **节点**：指一台运行着 Shadowsocks 或 V2Ray 代理服务的 服务器。

### 计算机与服务器的区别：
- **计算机（电脑）**：像一辆私家车，为个人设计，随时开，随时停，主要服务于一个人。
- **服务器**：像重型货车，为特定任务设计，注重载重能力和持续运行时间，服务大量请求。

这两者本质上都是“**算力载体**”，所有电子设备，都是这种相同的本质。


- **虚拟主机（Shared Hosting）**：
  - **资源共享**，性能不稳定，难以满足 VPN 高并发需求。
  - **不支持 root 权限**，无法自定义配置。
  - **不适合搭建 VPN**，一般不支持高级网络配置和端口转发。
  - 用户只能使用提供的控制面板（如cPanel、Plesk）进行操作，无法获得root/管理员权限。



- **VPS（Virtual Private Server）**：
  - **独立资源**，稳定性较好，适合中小型 VPN 服务。
  - **提供 root 权限**，可以自由配置 VPN 软件。
  - **适合搭建 VPN**，能够支持自定义端口、加密等配置。
  - 用户拥有完整的系统权限（root或管理员），可以安装任意软件，自定义配置环境。

- **云服务器（Cloud Server）**：
  - **弹性资源**，可以根据需求扩展，支持大规模并发。
  - **高可用性和负载均衡**，适合高流量 VPN 服务。
  - **完全可自定义**，支持全权限配置和自动扩展。
  - **适合搭建大型 VPN 服务**，但价格较高，技术要求更高。


Ubuntu
一个开源免费的 Linux 操作系统，类似于 Windows 或 macOS，但是它是 开源、免费的，常用于服务器是服务器安转功能性应用的基础。

代理协议
在客户端和服务器之间转发和加密数据的通信协议，用于绕过网络限制、提高安全性或隐私的工具

常见的代理协议包括：

Shadowsocks：轻量级、快速、易于配置的代理协议，常用于翻墙。
V2Ray：功能更强大的代理协议，支持多种协议（如 VMess），提供更高的安全性和灵活性。
Socks5：一种常见的代理协议，支持各种类型的流量转发，包括TCP和UDP。
HTTP/HTTPS：基于HTTP的代理协议，常用于浏览网页，支持加密传输。
Trojan：基于 HTTPS 的代理协议，具有较强的隐蔽性和防封锁能力。


**代理协议的关键作用**
**加密数据(绕过网络限制)**：确保请求和响应的安全性。

代理客户端
iOS上安装的“Quantumult-X”、“小火箭”等应用

### 代理工作流程

1. **客户端 (手机或电脑)**  
   发出请求（打开网页或使用应用）。  
   ↓
2. **代理客户端 (Quantumult-X)**  
   收到请求，并通过【代理协议】 **加密** 请求数据，以**绕过网络限制**：帮助访问被封锁的内容。  
   ↓
3. **代理服务器(节点服务器)**  
   解密请求并转发到目标服务器。  
   ↓
4. **目标服务器(例如，Google, YouTube)**  
   返回响应数据。  
   ↓
7. **代理服务器(节点服务器)**  
   将加密的响应数据返回给客户端。  
   ↓
8. **客户端(手机或电脑)**  
   解密数据并显示网页或应用内容。
